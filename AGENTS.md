# AGENTS.md

このファイルは、MangaViewer リポジトリで作業するエージェント向けの軽量ガイドです。

## 1. スコープ
- この `AGENTS.md` の適用範囲は **リポジトリ全体**（`/workspace/MangaViewer` 配下）です。
- より深い階層に別の `AGENTS.md` がある場合は、そちらを優先します。

## 2. 基本方針（軽量）
- 既存の設計・命名・構成に合わせ、**最小差分**で対応する。
- 迷う場合は、`doc/README.md` と既存実装（`src/`）の整合性を優先する。
- 大規模リファクタは依頼がない限り行わない。

## 3. リポジトリ構成（現状）
- 実装: `src/`
  - `domain/`, `application/`, `infrastructure/`, `controller/`, `views/` のレイヤ構成
- 起動スクリプト: `scripts/start-server.js`
- ドキュメント: `doc/`
- テスト: `__tests__/`
  - `small` / `medium` / `large(e2e)` の区分

## 4. よく使う npm scripts
- 開発起動: `npm run dev`
- 本番同等起動: `npm run start`
- テスト起動用サーバー: `npm run start:test`
- 単体/統合テスト: `npm run test:small`, `npm run test:medium`
- E2E: `npm run test:e2e`
- 全テスト: `npm run test:all`

## 5. 命名・ドキュメント整合
`doc/README.md` にある命名方針に従うこと。

- 同一ユースケースの `doc/`・`src/`・`__tests__/` は basename をできるだけ揃える。
- アプリケーションサービスの設計書名は、実装名に合わせて `*Service` を含める。
- 名称変更時は、見出し・図・参照リンク・テストケース名の名称ゆれを残さない。

## 6. 変更時の実務ルール
- 1つの変更に目的を混在させない（機能追加とリファクタは分ける）。
- 依頼がない限り、公開インターフェースや API 契約を壊さない。
- セキュリティ関連（認証・セッション・環境変数）は既存 fail-close 方針を維持する。

## 7. コミットメッセージ規約
- 日本語で記述する。
- 1行目: `プレフィックス: コミットによって達成する要件、仕様`
  - 使用可能プレフィックス: `docs`, `feat`, `fix`, `refactor`, `style`, `test`
- 2行目: 空行
- 3行目以降: 変更内容の詳細、特に **why（なぜ必要か）** を記述する。

例:

```text
docs: AGENTS.mdに軽量運用ガイドを追加

新規参加者やエージェントが既存構成に沿って最小差分で作業できるよう、
リポジトリ固有の構成・命名方針・主要スクリプトを明文化したため。
```

## 8. プルリクエスト本文の目安
- 目的（何を解決するか）
- 変更点（何をどう変えたか）
- 影響範囲（どこに影響するか）
- 確認手順（レビューアが再現する方法）

以上。厳密すぎる制約は置かず、運用で不足が見えたら随時更新する。
