# 未定義ルート共通 404 ハンドラー テスト観点（medium）

## medium テスト観点
- [既存ルートが先に優先され、未定義ルートのみ 404 ハンドラーが処理する](#既存ルートが先に優先され未定義ルートのみ-404-ハンドラーが処理する)
- [認証要否にかかわらず未定義ルートでは共通の 404 JSON を返す](#認証要否にかかわらず未定義ルートでは共通の-404-json-を返す)

---

### 既存ルートが先に優先され未定義ルートのみ 404 ハンドラーが処理する
- **前提**
  - screen / api の既存ルートが登録されている。
- **操作**
  - 既存ルートと未定義ルートへそれぞれリクエストする。
- **結果**
  - 既存ルートは個別仕様のレスポンスを返す。
  - 未定義ルートだけが 404 ハンドラーで `{ message: 'Not Found' }` を返す。

---

### 認証要否にかかわらず未定義ルートでは共通の 404 JSON を返す
- **前提**
  - 認証が必要な画面/API と不要な画面/API が混在している。
- **操作**
  - それぞれの系統で未定義パスへアクセスする。
- **結果**
  - 認証ミドルウェアの対象外となり、共通の 404 JSON 応答になる。
