# メディア検索

## 概要
ユーザーが検索条件を入力し、それに一致するメディアを一覧表示する

## アクター
- ユーザー
- 管理者

## 事前条件
- ユーザーがログインしている
- 検索画面が表示されている

## 検索条件

|    項目名    |     デフォルト      |                                        備考                                         |
| ------------ | ------------------- | ----------------------------------------------------------------------------------- |
| タイトル     | なし                |                                                                                     |
| タグ         | なし                | 複数指定可                                                                          |
| ソート方法   | 登録日新しい順      | 登録日新しい順 / 登録日古い順 / タイトル順 / タイトル逆順 / ランダム               |
| 取得開始位置 | 1                   | 必須。自然数のみ。 `/screen/summary` では `start` クエリとして受け取り、優先適用する |
| 取得数       | 20                  | 必須。自然数のみ。 `/screen/summary` では `size` クエリとして受け取る               |
| 一覧ページ   | 1                   | `summaryPage`。`start` 未指定時に `size` と組み合わせて開始位置を補完する           |

## 事後条件
- 検索条件に一致したメディアを一覧で画面に表示する
- 検索条件を画面に表示する
- ソート方法を画面に表示する
- 取得開始位置・取得数を維持したまま並び替えやページ遷移ができる

## トリガー
- ユーザーが検索条件を指定して検索を行う

## 基本フロー
1. ユーザーはタイトル・タグ・取得開始位置・取得数・並び順を入力して検索を実行する
2. システムは `start` / `size` を正規化し、必要に応じて `summaryPage` から開始位置を補完する
3. システムは検索条件に一致するメディアを取得する
4. システムはメディアを一覧にして画面に表示する
5. ユーザーは任意のメディアを選択して詳細画面に遷移する

## 代替フロー
- 検索条件に一致するメディアが存在しない場合は「検索結果が0件」のメッセージを表示する
- `start` / `size` / `summaryPage` に不正値が渡された場合は既定値で補完する

## 拡張フロー
- ユーザーがソート方法を指定する
  - 登録日時順、更新日時順、ランダム、etc...

## 関連ユースケース
- メディア一覧表示（事後）
