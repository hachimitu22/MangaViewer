# ユーザー集約

## ユーザーの生成に成功する
- 前提
  - 有効なユーザーIDが与えられている。
- 操作
  - ユーザーを生成する。
- 結果
  - ユーザーが生成される。
  - ユーザーは指定したユーザーIDを保持する。
  - お気に入りは空である。
  - あとで見るは空である。

## ユーザーの生成に失敗する
- 前提
  - 無効なユーザーIDが与えられている。
- 操作
  - ユーザーを生成する。
- 結果
  - ユーザーが生成されない。

## メディアをお気に入りに追加できる
- 前提
  - お気に入りを持たないのユーザーが存在する。
- 操作
  - メディアをお気に入りに追加する。
- 結果
  - メディアがお気に入りにが追加される。
  - あとで見るは空である。

## メディア以外はお気に入りに追加できない
- 前提
  - お気に入りを持たないのユーザーが存在する。
- 操作
  - メディア以外をお気に入りに追加する。
- 結果
  - メディアがお気に入りに追加されない。

## お気に入りに追加されたメディアはお気に入りに追加できない
- 前提
  - お気に入りを持つユーザーが存在する。
- 操作
  - 同一メディアをお気に入りに追加する。
- 結果
  - お気に入りは追加されない。
  - お気に入り内に同一メディアIDが重複しないという不変条件が保たれる。
  - あとで見るは空である。

## お気に入りは登録順を保持する
- 前提
  - お気に入りを持たないユーザーが存在する。
- 操作
  - 複数のお気に入りを追加する。
- 結果
  - お気に入りの順序は追加した順序となる。

## お気に入りを削除できる
- 前提
  - お気に入りを持つユーザーが存在する。
- 操作
  - お気に入りのメディアを削除する。
- 結果
  - お気に入りが削除される。
  - あとで見るは空である。

## お気に入りに追加していないメディアをお気に入りから削除できない
- 前提
  - お気に入りを持つユーザーが存在する。
- 操作
  - お気に入りに存在しないメディアを削除する。
- 結果
  - お気に入りの削除に失敗する。
