# IMediaRepository

## 概要
メディア永続化の抽象契約。

---

## トランザクション境界の方針
- 境界は Application Service の `execute` で開始する。
- Repository は引数で受け取らず、実行コンテキストから取得して利用する。
- Application 層ではインフラ依存語彙を避け、`transaction` という語を使わない。

---

## テスト方針
- interface 自体はテスト対象にせず、`SequelizeMediaRepository` のような具象実装で保存・更新・削除・未登録ID・異常系を検証する。
- 文書のテストケースも interface ではなく具象実装／契約テストに寄せ、トランザクション連携は具象実装の責務として扱う。
- 実装が複数になった場合は、「`Media` を保存できる」「未登録IDで `null` を返す」といった契約テストを共通化する。

---

## 機能
### 保存
メディアを永続化する。

#### 入力
- メディア

#### 出力
- なし

---

### メディアIDからメディアを取得
リポジトリからメディアIDでメディアを取得する。
メディアIDと一致するメディアが存在しない場合は何も返さない。

#### 入力
- メディアID

#### 出力
- メディア
