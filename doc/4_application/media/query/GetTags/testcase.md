# GetTags テストケース

## 正常系

### 登録済みタグ一覧を取得できる

- **前提**
  - タグが複数件登録されている。
  - Repository は全タグ一覧を正常に返却する。
    - 例：
      - カテゴリー: "ジャンル", ラベル: "SF"
      - カテゴリー: "ジャンル", ラベル: "ファンタジー"
      - カテゴリー: "年齢制限", ラベル: "R15"

- **操作**
  - GetTags を実行する。

- **結果**
  - タグ一覧が取得できる。
  - 取得結果にすべての登録済みタグが含まれる。
  - 取得されたタグは TagOutput（カテゴリー名・ラベル名）形式で返却される。

---

### カテゴリー別にタグ一覧を取得できる

- **前提**
  - 以下のタグが登録されている。
    - カテゴリー: "ジャンル", ラベル: "SF"
    - カテゴリー: "ジャンル", ラベル: "ファンタジー"
    - カテゴリー: "年齢制限", ラベル: "R15"
  - Repository は指定カテゴリーのタグのみ取得可能である。

- **操作**
  - カテゴリー名 "ジャンル" を指定して GetTags を実行する。

- **結果**
  - カテゴリーが "ジャンル" のタグのみが取得される。
  - 取得結果に "SF" と "ファンタジー" が含まれる。
  - "年齢制限" カテゴリーのタグは含まれない。
  - 取得されたタグは TagOutput 形式で返却される。

---

### 指定カテゴリーにタグが存在しない場合は空配列を返す

- **前提**
  - 以下のタグが登録されている。
    - カテゴリー: "ジャンル", ラベル: "SF"
  - Repository は指定カテゴリーに該当するタグが存在しない場合、空配列を返す。

- **操作**
  - 存在しないカテゴリー名 "制作会社" を指定して GetTags を実行する。

- **結果**
  - エラーにはならない。
  - 空配列が返却される。
  - タグ一覧の件数は 0 件である。

---

## 異常系

### Repository 取得に失敗した場合は失敗する

- **前提**
  - Repository がタグ一覧取得時に例外またはエラーを返す。

- **操作**
  - GetTags を実行する。

- **結果**
  - GetTags は失敗する。
  - エラーが呼び出し元に返却される。
  - タグ一覧は返却されない。
  - 永続化層のエラーがアプリケーションエラーとして扱われる。
