# SearchMedia テストケース

## 正常系

- 条件に一致するメディア一覧を取得できる
  - 前提:
    - 検索条件に一致するメディアが複数存在する
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - MediaOverview DTO の配列が返る
    - 件数が一致している
    - 各要素が表示用DTOでありドメイン型を含まない

- 複数条件で検索できる
  - 前提:
    - タグ・カテゴリーを含む検索条件を指定する
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - すべての条件を満たすメディアのみ返る

- ソート条件を指定して取得できる
  - 前提:
    - ソート条件（例: 作成日降順）を指定する
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - 指定された順序で結果が返る

- ページング条件を指定して取得できる
  - 前提:
    - page, size を指定する
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - 指定ページ分の結果のみ返る

- 検索結果が0件でも空配列を返す
  - 前提:
    - 条件に一致するメディアが存在しない
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - 空配列が返る
    - エラーは発生しない

- ページング範囲外でも空配列を返す
  - 前提:
    - 総件数を超えるページ番号を指定する
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - 空配列が返る
    - エラーは発生しない

## 境界・仕様確認

- 最大取得件数制限が適用される
  - 前提:
    - size に最大値を超える数値を指定する
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - 上限値までの件数で返る

- ソート未指定時はデフォルト順で返る
  - 前提:
    - ソート条件を指定しない
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - 定義済みデフォルト順で返る

- ページング情報が正しく返る
  - 前提:
    - ページング条件を指定する
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - 総件数・現在ページ・総ページ数が正しい

## 異常系

- 不正なページング条件の場合は取得できない
  - 前提:
    - page < 0 や size <= 0 を指定する
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - 失敗結果が返る

- Repository 取得に失敗した場合は失敗する
  - 前提:
    - ReadRepository が例外を返す
  - 操作:
    - SearchMediaQuery を実行する
  - 期待結果:
    - Query が失敗する
