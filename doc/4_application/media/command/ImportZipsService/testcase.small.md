# ImportZipsService テスト観点（small）

## このドキュメントの位置づけ（`readme.md` との責務分担）
- `readme.md`: `ImportZipsService` の業務ルール・入出力・失敗時の扱いを定義する。
- `testcase.small.md`（本書）: **純粋ロジック**（入力検証・並び順決定・結果種別集計・reasonCode 決定）を、依存を置き換えた単体テストで保証する。
- `testcase.medium.md`: 実ファイル I/O・実トランザクションを含む統合観点を保証する。

---

## small の対象範囲（実ファイル I/O 依存の削減）
- 本書の small では、以下の外部依存を **スタブ/フェイク化** する。
  - zip 列挙器（対象 zip 一覧取得）
  - zip 展開器（画像/非画像の列挙、展開結果）
  - リポジトリ（メディア永続化）
  - トランザクション（begin/commit/rollback）
- 実ファイルシステム、実 DB には依存しない。
- 検証は「呼び出し結果（戻り値）」「依存呼び出し記録（引数・回数）」「集計値」のみに限定する。

### 依存の置き換え方（テストダブル方針）
1. **zip 列挙器（スタブ）**
   - 事前に定義した zip 名配列を返すだけのスタブを用いる。
   - 0 件/上限境界/自然順比較用の入力を、ファイル作成なしで注入する。

2. **zip 展開器（スタブ）**
   - zip ごとに「画像エントリ一覧」「非画像一覧」「サイズ情報」「失敗 reasonCode」を返すスタブを用いる。
   - `NO_IMAGES` / `ZIP_IMAGE_COUNT_LIMIT_EXCEEDED` / `IMAGE_FILE_SIZE_LIMIT_EXCEEDED` を、I/O 例外ではなく業務ルール入力として再現する。

3. **リポジトリ（フェイク）**
   - インメモリで保存要求を記録するフェイクを用いる。
   - 保存済み件数・保存順・失敗 zip で保存されないこと（zip 単位原子性）を検証する。

4. **トランザクション（フェイク）**
   - begin/commit/rollback の呼び出し履歴を保持するフェイクを用いる。
   - 成功 zip は commit、失敗 zip は rollback が選択されることのみを確認する（実 DB ロールバックは medium 範囲）。

## small 観点（純粋ロジック）

### 1) zip名 → タイトル変換
**前提**
- `foo-bar.zip` を zip 列挙スタブで入力する。

**操作**
- `ImportZipsService` を実行し、リポジトリフェイクへの保存要求を確認する。

**期待結果**
- タイトルは zip 名から `.zip` を除去した値で作成される。
- 取り込み結果の `zip名` は元ファイル名を保持し、タイトル生成規則と混同しない。

### 2) 同名タイトルの重複許容
**前提**
- 同名タイトルに相当する zip を複数入力する（または既存タイトルあり状態をリポジトリフェイクで再現する）。

**操作**
- 当該 zip 群を取り込む。

**期待結果**
- タイトル重複を理由に失敗しない。
- zip 単位結果は、それぞれ独立して成功/失敗が判定される。

### 3) 画像のみ採用・非画像無視
**前提**
- zip 展開スタブが、画像（jpg/png/webp 等）と非画像（txt/json/db 等）を混在返却する。

**操作**
- 当該 zip を取り込む。

**期待結果**
- 画像ファイルのみが取り込み対象になる。
- 非画像ファイルは `無視ファイル一覧` に記録され、取り込み失敗の直接原因にしない。

### 4) ソート順（`1,2,10` / 大小文字混在）
**前提**
- zip 展開スタブが `1.png`, `2.png`, `10.png`, `A.jpg`, `a.jpg` を返す。

**操作**
- 取り込み後のコンテンツ順序（保存要求順）を確認する。

**期待結果**
- 数値を含むファイル名は自然順で `1,2,10` となる。
- 大小文字混在時も自然順を第一キーとして扱い、同順位時はファイル名辞書順の第二キーで順序が決定される。

### 5) 画像0件 zip は失敗
**前提**
- zip 展開スタブが非画像のみ（= 画像 0 件）を返す。

**操作**
- 当該 zip を取り込む。

**期待結果**
- その zip の結果は `FAILED` になる。
- `reasonCode` は `NO_IMAGES` と完全一致で判定する。
- `reasonDetail` は存在/非存在の許容範囲で検証する（補助文字列の部分一致は任意）。
- 失敗 zip に対する保存要求は発行されない（zip 単位原子性）。

### 6) 全成功時は `SUCCESS` を返す
**前提**
- 取り込み対象 zip がすべて成功シナリオのスタブ応答である。

**操作**
- `ImportZipsService` を実行し、`実行結果種別` を確認する。

**期待結果**
- `実行結果種別=SUCCESS` を返す。

### 7) 一部失敗時は `PARTIAL_FAILURE` を返す
**前提**
- 成功 zip と失敗 zip（例: 画像 0 件）が混在するスタブ応答を用意する。

**操作**

**期待結果**

### 8) 対象 zip 0 件の戻り値
**前提**

**操作**
- `ImportZipsService` を実行する。
