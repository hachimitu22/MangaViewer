## small 観点

### 1) zip名 → タイトル変換
**前提**
- `foo-bar.zip` のように拡張子付き zip 名を持つ入力を用意する。

**操作**
- `ImportZipsService` を実行し、登録されるメディアタイトルを確認する。

**期待結果**
- タイトルは zip 名から `.zip` を除去した値で作成される。
- 取り込み結果の `zip名` は元ファイル名を保持し、タイトル生成規則と混同しない。

### 2) 同名タイトルの重複許容
**前提**
- タイトル変換後に同名となる zip（例: 同一 basename を持つ別配置 zip）を複数用意する。

**操作**
- 複数 zip を同一実行で取り込む。

**期待結果**
- タイトル重複を理由に失敗しない。
- zip 単位結果は、それぞれ独立して成功/失敗が判定される。

### 3) 画像のみ採用・非画像無視
**前提**
- 画像ファイル（jpg/png/webp 等）と非画像ファイル（txt/json/db 等）が混在する zip を用意する。

**操作**
- 当該 zip を取り込む。

**期待結果**
- 画像ファイルのみが取り込み対象になる。
- 非画像ファイルは `無視ファイル一覧` に記録され、取り込み失敗の直接原因にしない。

### 4) ソート順（`1,2,10` / 大小文字混在）
**前提**
- `1.png`, `2.png`, `10.png` と、`A.jpg`, `a.jpg` のような比較ケースを同一 zip に含める。

**操作**
- 取り込み後のコンテンツ順序（保存順または参照順）を確認する。

**期待結果**
- 数値を含むファイル名は自然順で `1,2,10` となる。
- 大小文字混在時も自然順を第一キーとして扱い、同順位時はファイル名辞書順の第二キーで順序が決定される。

### 5) 画像0件 zip は失敗
**前提**
- 非画像ファイルのみを含む zip（= 画像 0 件）を用意する。

**操作**
- 当該 zip を取り込む。

**期待結果**
- その zip の結果は `FAILED` になる。
- `理由` に、画像 0 件で取り込み不可であることが判別可能な情報が含まれる。
- 失敗 zip に対する永続化は行われない（zip 単位原子性）。

### 6) ユースケース結果種別の判定（全成功 / 一部失敗 / 入力不正）
**前提**
- 以下 3 パターンの入力条件を用意する。
  - A: 全 zip が正常取り込み可能。
  - B: 成功 zip と失敗 zip が混在。
  - C: 対象フォルダ未指定や上限値不正など、入力契約を満たさない。

**操作**
- 各パターンで `ImportZipsService` を実行し、`実行結果種別` と `全体サマリ` を確認する。

**期待結果**
- A（全成功）: `実行結果種別=SUCCESS`。
- B（一部失敗）: `実行結果種別=PARTIAL_FAILURE`。
- C（入力不正）: `実行結果種別=INVALID_INPUT`（または契約上同等の失敗応答）を返し、異常系が検知できる。

> 補足: 数値終了コード（`0` / `1` / `2`）への変換、および実行前検証エラーの `2` 判定は CLI テストで検証する。
