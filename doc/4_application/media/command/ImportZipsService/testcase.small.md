## small 観点

### 1) zip名 → タイトル変換
**前提**
- `foo-bar.zip` のように拡張子付き zip 名を持つ入力を用意する。

**操作**
- `ImportZipsService` を実行し、登録されるメディアタイトルを確認する。

**期待結果**
- タイトルは zip 名から `.zip` を除去した値で作成される。
- 取り込み結果の `zip名` は元ファイル名を保持し、タイトル生成規則と混同しない。

### 2) 同名タイトルの重複許容
**前提**
- タイトル変換後に同名となる zip（例: 同一 basename を持つ別配置 zip）を複数用意する。

**操作**
- 複数 zip を同一実行で取り込む。

**期待結果**
- タイトル重複を理由に失敗しない。
- zip 単位結果は、それぞれ独立して成功/失敗が判定される。

### 3) 画像のみ採用・非画像無視
**前提**
- 画像ファイル（jpg/png/webp 等）と非画像ファイル（txt/json/db 等）が混在する zip を用意する。

**操作**
- 当該 zip を取り込む。

**期待結果**
- 画像ファイルのみが取り込み対象になる。
- 非画像ファイルは `無視ファイル一覧` に記録され、取り込み失敗の直接原因にしない。

### 4) ソート順（`1,2,10` / 大小文字混在）
**前提**
- `1.png`, `2.png`, `10.png` と、`A.jpg`, `a.jpg` のような比較ケースを同一 zip に含める。

**操作**
- 取り込み後のコンテンツ順序（保存順または参照順）を確認する。

**期待結果**
- 数値を含むファイル名は自然順で `1,2,10` となる。
- 大小文字混在時も自然順を第一キーとして扱い、同順位時はファイル名辞書順の第二キーで順序が決定される。

### 5) 画像0件 zip は失敗
**前提**
- 非画像ファイルのみを含む zip（= 画像 0 件）を用意する。

**操作**
- 当該 zip を取り込む。

**期待結果**
- その zip の結果は `FAILED` になる。
- `理由` は文字列部分一致ではなく分類識別子で判定し、期待値は `NO_IMAGES` とする（任意詳細の有無は許容）。
- 失敗 zip に対する永続化は行われない（zip 単位原子性）。

### 6) 全成功時は `SUCCESS` を返す
**前提**
- 取り込み対象の zip がすべて正常取り込み可能である。

**操作**
- `ImportZipsService` を実行し、`実行結果種別` を確認する。

**期待結果**
- `実行結果種別=SUCCESS` を返す。

### 7) 一部失敗時は `PARTIAL_FAILURE` を返す
**前提**
- 成功 zip と失敗 zip（例: 画像 0 件）が混在する。

**操作**
- `ImportZipsService` を実行し、`実行結果種別` を確認する。

**期待結果**
- `実行結果種別=PARTIAL_FAILURE` を返す。

### 8) 対象 zip 0 件の戻り値
**前提**
- 対象フォルダ配下に zip が 1 件も存在しない入力を用意する。

**操作**
- `ImportZipsService` を実行する。

**期待結果**
- `zip単位結果一覧=[]`（空配列）を返す。
- `全体サマリ` は `対象zip件数=0 / 成功件数=0 / 失敗件数=0` を返す。
- `実行結果種別=SUCCESS` を返す。



### 9) `zipごとの最大画像数` が `上限-1` のとき許可される
**前提**
- `zipごとの最大画像数=MAX_IMAGES_PER_ZIP` を設定する。
- 画像枚数が `MAX_IMAGES_PER_ZIP-1` の zip を用意する。

**操作**
- 当該 zip を `ImportZipsService` に入力し、zip 単位結果を確認する。

**期待結果**
- `SUCCESS`（許可）になる。
- 境界値判定は純粋ロジックとして検証し、I/O 失敗やロールバック観点は medium で扱う。
