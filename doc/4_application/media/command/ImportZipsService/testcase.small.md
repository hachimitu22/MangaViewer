# ImportZipsService テスト観点（small）

## このドキュメントの位置づけ（`readme.md` との責務分担）
- `readme.md`: `ImportZipsService` の業務ルール・入出力・失敗時の扱いを定義する。
- `testcase.small.md`（本書）: **純粋ロジック**（入力検証・並び順決定・結果種別集計・reasonCode 決定）を、依存を置き換えた単体テストで保証する。
- `testcase.medium.md`: 実ファイル I/O・実トランザクションを含む統合観点を保証する。

---

## small の対象範囲（実ファイル I/O 依存の削減）
- 本書の small では、以下の外部依存を **スタブ/フェイク化** する。
  - zip 列挙器（対象 zip 一覧取得）
  - zip 展開器（画像/非画像の列挙、展開結果）
  - リポジトリ（メディア永続化）
  - トランザクション（begin/commit/rollback）
- 実ファイルシステム、実 DB には依存しない。
- 検証は「呼び出し結果（戻り値）」「依存呼び出し記録（引数・回数）」「集計値」のみに限定する。

## ケース設計ルール（切り分け優先）
- 1 ケース 1 主要アサーションとする。
- 副次アサーションは最小化し、別観点で再利用しない。
- 前提データはケースごとに最小構成とする。
- ケース名は期待振る舞いを読める文で統一する。

### 依存の置き換え方（テストダブル方針）
1. **zip 列挙器（スタブ）**
   - 事前に定義した zip 名配列を返すだけのスタブを用いる。
   - 0 件/上限境界/自然順比較用の入力を、ファイル作成なしで注入する。

2. **zip 展開器（スタブ）**
   - zip ごとに「画像エントリ一覧」「非画像一覧」「サイズ情報」「失敗 reasonCode」を返すスタブを用いる。
   - `NO_IMAGES` / `ZIP_IMAGE_COUNT_LIMIT_EXCEEDED` / `IMAGE_FILE_SIZE_LIMIT_EXCEEDED` を、I/O 例外ではなく業務ルール入力として再現する。

3. **リポジトリ（フェイク）**
   - インメモリで保存要求を記録するフェイクを用いる。
   - 保存済み件数・保存順・失敗 zip で保存されないこと（zip 単位原子性）を検証する。

4. **トランザクション（フェイク）**
   - begin/commit/rollback の呼び出し履歴を保持するフェイクを用いる。
   - 成功 zip は commit、失敗 zip は rollback が選択されることのみを確認する（実 DB ロールバックは medium 範囲）。

## small 観点（純粋ロジック）

### 1) zip名からタイトルを生成する
- **主要アサーション**
  - タイトルは zip 名から `.zip` を除去した値で生成される。
- **前提（最小構成）**
  - zip 列挙スタブに `foo-bar.zip` の 1 件を設定する。
- **操作**
  - `ImportZipsService` を実行し、保存要求を確認する。
- **確認（最小）**
  - タイトルが `foo-bar` で保存要求される。

### 2) 取り込み結果のzip名に元ファイル名を保持する
- **主要アサーション**
  - 取り込み結果の `zip名` は元ファイル名を保持する。
- **前提（最小構成）**
  - zip 列挙スタブに `foo-bar.zip` の 1 件を設定する。
- **操作**
  - `ImportZipsService` を実行し、zip 単位結果を確認する。
- **確認（最小）**
  - `zip名=foo-bar.zip` が返る。

### 3) 同名タイトルでも重複を理由に失敗しない
- **主要アサーション**
  - 同名タイトルを含む入力でもタイトル重複を理由に失敗しない。
- **前提（最小構成）**
  - 同名タイトルに相当する zip を複数設定する（または既存タイトルありをフェイクで再現）。
- **操作**
  - 当該 zip 群を取り込む。
- **確認（最小）**
  - タイトル重複由来の失敗が発生しない。

### 4) 非画像ファイルを取り込み対象から除外する
- **主要アサーション**

- **前提（最小構成）**

- **操作**

- **確認（最小）**
  - 保存要求に非画像が含まれない。
