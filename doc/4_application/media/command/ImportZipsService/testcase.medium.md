# ImportZipsService テストケース（medium: 実 I/O 統合）

## 目的
- medium は、`ImportZipsService` と **実ファイル I/O（FS）・実トランザクション** の結合点を検証する。
- small と同一観点の重複を避け、1 ケース 1 責務で「どちらで担保するか」を明確化する。
- ログ出力（`AppLogger` による人間可読ログ/JSONL整形・出力）は CLI / infrastructure の責務であり、本書の対象外とする。

## small / medium の責務分担（重複整理）
| 観点 | small で担保 | medium で担保 |
|---|---|---|
| 入力検証（契約違反、上限判定の境界） | 実施（純粋ロジック） | 原則実施しない（I/O 副作用有無の確認が必要な最小ケースのみ） |
| 並び順決定（zip名/画像名の自然順） | 実施（決定ロジック） | 同一入力で処理順が安定し、実行ごとの差異が出ないことを実 I/O で確認 |
| 結果種別集計（SUCCESS/PARTIAL_FAILURE/INVALID_INPUT*） | 実施（集計ロジック） | 実 I/O を伴う複数 zip 実行で集計と実副作用が整合することを確認 |
| reasonCode 決定 | 実施（`NO_IMAGES` 等の分類） | 実 I/O を伴う失敗入力で reasonCode が設計どおり返ることを確認 |
| zip 単位原子性 / ロールバック | 対象外（呼び出し選択のみ） | 実施（失敗 zip の副作用が残らない） |
| ログ出力（人間可読/JSONL） | 対象外 | 対象外（CLI / infrastructure のテストで担保） |

## ケース詳細

### 対象フォルダに zip が 0 件でも正常終了する

- **主要アサーション**
  - zip 0 件入力で異常終了せず `SUCCESS` を返す。
- **前提（最小構成）**
  - 実ファイル I/O で空の対象フォルダを 1 つ用意する（直下に zip なし）。
- **操作**
  - ImportZipsService を 1 回実行する。
- **確認（最小）**
  - `実行結果種別=SUCCESS`。

---

### 失敗 zip があっても後続 zip を処理する

- **主要アサーション**
  - 先行 zip が失敗しても後続 zip の処理を継続する。
- **前提（最小構成）**
  - 実ファイル I/O で 2 件の zip を配置する。
    - 1 件目: 失敗する zip（例: 画像 0 件）
    - 2 件目: 成功する zip
- **操作**
  - ImportZipsService を 1 回実行する。
- **確認（最小）**
  - zip 単位結果一覧で 2 件目にも結果が記録される。

---

### 成功 1 件・失敗 1 件を処理した件数を正しく集計する

- **主要アサーション**
  - 成功 1 件・失敗 1 件の入力で全体サマリ件数が一致する。
- **前提（最小構成）**
  - 実ファイル I/O で 2 件の zip を配置する。
    - 成功する zip: 1 件
    - 失敗する zip: 1 件（例: 画像 0 件）
- **操作**
  - ImportZipsService を 1 回実行する。
- **確認（最小）**
  - `全体サマリ=対象zip件数2 / 成功件数1 / 失敗件数1`。

---

### 成功 1 件・失敗 1 件のとき結果種別を PARTIAL_FAILURE と判定する

- **主要アサーション**
  - 成功と失敗が混在する実行で `PARTIAL_FAILURE` を返す。
- **前提（最小構成）**
  - 実ファイル I/O で 2 件の zip を配置する。
    - 成功する zip: 1 件
    - 失敗する zip: 1 件（例: 画像 0 件）
- **操作**
  - ImportZipsService を 1 回実行する。
- **確認（最小）**
  - `実行結果種別=PARTIAL_FAILURE`。

---

### 失敗 zip では永続化と作業ファイルの副作用を残さない

- **主要アサーション**
  - 失敗 zip に対応する副作用（永続化・作業ファイル）が残らない。
- **前提（最小構成）**
  - 実ファイル I/O で失敗する zip を 1 件だけ配置する（例: 展開途中で失敗する内容）。
- **操作**
  - ImportZipsService を 1 回実行する。
- **確認（最小）**
  - 失敗 zip に対応する永続化レコードが存在しない。
  - 失敗 zip に対応する一時展開物・中間生成物が作業領域に残らない。

---

### 同一入力では zip の処理順が安定する

- **主要アサーション**
  - 同一入力で複数回実行しても zip の処理順が一致する。
- **前提（最小構成）**
  - 実ファイル I/O で、比較順が揺れやすい zip 名を配置する。
    - 例: `vol1.zip`, `vol02.zip`, `VOL2.zip`, `ＶＯＬ３.zip`, `vol10.zip`
  - 同一入力で 2 回以上実行できる状態にする。
- **操作**
  - ImportZipsService を同一入力で複数回実行し、各回の zip 単位結果一覧の順序を取得する。
- **確認（最小）**
  - すべての実行回で zip の処理順が一致する。

---

### 上限超過入力は実行全体を拒否し副作用を残さない

- **主要アサーション**
  - 上限超過時は zip ごとの処理を開始せず副作用を残さない。
- **前提（最小構成）**
  - `1実行あたり最大zip処理件数=MAX_ZIPS_PER_RUN` を設定する。
  - 実ファイル I/O で zip 件数 `MAX_ZIPS_PER_RUN+1` を用意する。
- **操作**
  - ImportZipsService を 1 回実行する。
- **確認（最小）**
  - `実行結果種別=INVALID_INPUT_RUN_ZIP_COUNT_LIMIT_EXCEEDED`。
  - 永続化・展開作業の副作用を残さない。

> 補足: 上限 `-1 / = / +1` の境界値そのものは small で担保し、medium は「拒否時に実副作用がないこと」の統合確認に限定する。
