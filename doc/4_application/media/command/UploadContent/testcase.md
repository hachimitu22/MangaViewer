# UploadContent テストケース詳細

## 正常系

### コンテンツを保存できる

- **前提**
  - 保存対象のコンテンツパスが正しい形式である。
  - 管理者として認証済みである。
  - インフラ層の保存処理が正常に動作する。

- **操作**
  - UploadContent に以下を指定して実行する。
    - コンテンツ一覧：1件（有効なパス）

- **結果**
  - コンテンツが保存される。
  - UploadContentOutput の 成否 が true を返す。
  - エラーは発生しない。

---

### 複数コンテンツをまとめて保存できる

- **前提**
  - 複数の有効なコンテンツパスが存在する。
  - 管理者として認証済みである。
  - すべてのコンテンツ保存処理が成功する。

- **操作**
  - UploadContent に複数件の ContentInput を指定して実行する。

- **結果**
  - すべてのコンテンツが保存される。
  - UploadContentOutput の 成否 が true を返す。
  - エラーは発生しない。

---

## 異常系

### コンテンツ保存に失敗した場合はエラーとなる

- **前提**
  - 保存対象のコンテンツパスが正しい。
  - 管理者として認証済みである。
  - インフラ層の保存処理が失敗する状態である（例：ストレージエラー）。

- **操作**
  - UploadContent に有効な ContentInput を指定して実行する。

- **結果**
  - エラーが返却される。
  - UploadContentOutput は返却されない、または 成否=false となる。
  - コンテンツは保存されない。

---

### 空のコンテンツ一覧を指定した場合はエラーとなる

- **前提**
  - 管理者として認証済みである。

- **操作**
  - UploadContent に空配列の コンテンツ一覧 を指定して実行する。

- **結果**
  - エラーが返却される。
  - コンテンツ保存処理は実行されない。

---

### 無効なパスが含まれている場合はエラーとなる

- **前提**
  - コンテンツ一覧に無効なパス（空文字・不正形式）が含まれている。
  - 管理者として認証済みである。

- **操作**
  - UploadContent に無効なパスを含む ContentInput を指定して実行する。

- **結果**
  - エラーが返却される。
  - コンテンツは保存されない。
  - 永続化処理は行われない。
