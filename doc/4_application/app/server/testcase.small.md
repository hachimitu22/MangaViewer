## small 観点（運用CLI: `npm run import -- <dir>`）

### 1) 全成功時は終了コード `0`
**前提**
- `ImportZipsService` の戻り値を `実行結果種別=SUCCESS` に固定する（CLI 層単体で依存をスタブ化）。

**操作**
- CLI の import コマンドを実行する。

**期待結果**
- プロセス終了コードは `0` になる。
- 失敗扱いのログや終了コード `1/2` 判定は行われない。

### 2) 一部失敗時は終了コード `1`
**前提**
- `ImportZipsService` の戻り値を `実行結果種別=PARTIAL_FAILURE` に固定する（CLI 層単体で依存をスタブ化）。

**操作**
- CLI の import コマンドを実行する。

**期待結果**
- プロセス終了コードは `1` になる。
- 実行自体は完了扱いで、実行前検証エラーコード `2` にはならない。

### 3) 入力不正時は契約どおり未実行終了（終了コード `2`）
**前提**
- 実行前検証エラーとなる入力（例: `<dir>` 未指定）を与える。

**操作**
- CLI の import コマンドを実行する。

**期待結果**
- zip 走査・登録処理（`ImportZipsService` 呼び出し）が開始されない。
- プロセス終了コードは `2` になる。
- 入力契約違反を示すメッセージを出力し、異常原因が一意に識別できる。
